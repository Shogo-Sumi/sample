2.開発者ツールで CSS に関して実行できる操作を検索エンジンで調べ、便利だと思ったものを 3 つ挙げなさい

リアルタイムスタイル編集 (Live Editing in Styles Panel): StylesパネルでCSSプロパティの値を直接変更したり、新規追加/削除可能。変更が即座にページに反映されるので、試行錯誤が速く、レイアウト崩れの原因特定に便利。例えば、marginを調整してボックスモデルの問題を即修正。
擬似クラスの強制適用 (Force Pseudo-Classes): Elementsタブの:stateボタンで:hover, :focus, :activeなどをトグル。マウス操作なしで状態をシミュレートできるため、インタラクティブCSSのデバッグ（例: ホバーエフェクト）が効率的。
CSS特定度と無効ルールの確認 (Specificity and Overridden Rules): Stylesパネルでセレクタの特定度を表示し、無効化されたルール（ストライクアウト）をハイライト。競合するCSSの原因を素早く特定でき、複雑なスタイルシートのトラブルシューティングに役立つ。


3.15.4-10.2 のアプリの body 要素に対し、元々 HTML および JS 内で利用していなかった Tailwind CSS のクラス (bg-rose-600 など何でも良い) を開発者ツールから追加すると変更が反映されないが、これは何故か調べなさい

ex02のbodyに対して、下記のような変更を行ったが変更されなかった。

変更前
min-h-screen bg-gradient-to-br from-gray-100 to-gray-200 py-8 px-4

変更後
min-h-screen bg-gradient-to-br from-blue-100 to-blue-200 py-8 px-4

・考察
ex02ではスタイルシートは下記のコマンドで作成している。

npx tailwindcss -i ./ex02/input.css -o ./ex02/style.css

よって、ソースコードが記載されているindex.htmlやindex.jsに記載されていないcssの定義は、syle.css内に作成されていないと考えられる。
よって、作成されていないcssの定義については、開発者画面上で変更しても反映されないと考えられる。
